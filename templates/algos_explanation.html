Introduction to Machine Learning

Classic Machine Learning

Machine Learning, dominated by artificial intelligence, is an area that allows predicting results without the code explicitly containing the answer. For the machine to learn by itself it needs two important things:

  1. A data model that will be used as a basis to calculate a result
  2. A learning algorithm adapted to the model
The big problem with machine learning is that there is no such thing as a magic algorithm. There are a lot of algorithms, more or less expensive, depending on the data we want to process and the use we want to make of it. The strength of this area and that most of the work is studying the data and understanding how to process it. Subsequently most of the algorithms are already implemented and you just must use them. There are three main types of algorithms for learning the model. The learning is one of the most important part because it allows to bring a better precision of prediction.
  1. Supervised Learning
  2. Unsupervised Learning
  3. Reinforcement Learning

Supervised Learning

When we talk about supervised learning, we can easily understand that the machine needs help from humans. In this case the Machine must train with data labeled at the input and predefined output data. This kind of learning makes it possible to solve problems of regression (output variable is a continuous value, it can take any value) and of classification (output variable is a discrete value, related to a class, the number of possibility is limited).

Unsupervised Learning

Unlike supervised learning, unsupervised learning gives the machine a lot more flexibility. In fact, the machine receives a non-labeled data set and does not know the output values. Like a human being left in an unknown environment without a guide, he will try to regroup the data, make subsets, to classify a potential future unknown data. To achieve these subsets, we use learning algorithms for clustering (grouping of data according to their similarities) and association (finding relationships according to the attributes of the data).

Reinforcement Learning

This kind of method is special, and it is not the most used due to its complexity. The algorithms of this method are based on a system of reward and punishment. Obviously, the aim of the algorithm is to obtain the best reward for a given objective. Most generally, the algorithm creates its own dataset by exploring a specific environment.

Support Vector Machine Algorithm (SVM)

The SVM algorithm is a machine learning algorithm with supervised learning used to meet classification needs. It belongs to the category of linear classifiers because it uses linear separation of data. It is much more efficient in classification, especially binary classification. The algorithm can also handle regression problems.

The goal of an SVM algorithm is to find an optimal boundary by maximizing the distance between training points and the boundary thanks to support vectors located on the limit of each class. It will be necessary to use a "one-vs-one" or "one-vs-all" trick to be able to use the SVM in the multi-class classification.

We can use the "kernel trick" if the data is not separable in a linear way. However, it sometimes happens that the data model is not necessarily representable in the form of two linearly separable classes. It is a matter of common sense and in this case, it will be necessary to choose another type of algorithm such as the decision tree.

K-Nearst Neighbors Algorithm (KNN)

KNN is also a supervised learning algorithm that can solve both classification and regression problems. It is very easy to implement and study. There are no ambiguous settings or tips. The hyperparameters mainly concern the definition of the weight of the features, the way in which the distance is calculated (Euclidean, manhattan or minkowski) and the algorithm used for the search for neighbors. Here are the steps to follow:

  1. First step, you have to observe the model and define a K according to the noise (not always obvious)
  2. Secondly, the algorithm will have to calculate the distances between the data to be classified and the others
  3. Then we select the K data closest to the one we want to determine
  4. Finally we vote to know the label or the value induced by our data

KNN is a powerful algorithm that is very easy to understand and use. It is much more reliable for classification problems, but it can be used for regression. In our case of tumor detection (benign / malignant classification) it turned out to be very efficient. You still have to be careful with this kind of algorithm because it is quite sensitive to noise. The other useful information is related to K. The lower the K, the more there is a risk of having an overfitting problem (over-trained: the model is trained too narrowly). Conversely, the higher the K the more there is the risk of underfitting (under-trained: the model is treated too broadly).

Random Forest Algorithm

Before talking about the random forest, we must look at the decision tree of which it is composed.

The decision tree uses the same principle as for managerial decision trees. Obviously, a manager will make a small decision tree depending on what he knows it will be quick and efficient for a small subject. The computer can do a lot better by having a lot more data.

The decision tree is a supervised algorithm because they need labels, but the target variable can be real (regression problem) or categorical (classification problem). Respectively we call regression trees and classification trees.

As a reminder, a tree is an undirected, acyclic graph. For our algorithm we still start from the root to go to a leaf but if we draw the tree nothing prevents us from starting from a leaf to find how we arrived at the conclusion (See the example below).

The principle of the decision tree is to decide at each level of the tree until you find the leaf that corresponds to our data. for each case we have a data set. For the root and the intermediate nodes, it is necessary to answer a question (generally binary) with one of the data. Depending on the answer, we go to one of the intermediate nodes of the next level and we start interrogating again the data with a new question. The procedure is repeated until we reach a leaf that will allow us to obtain the result. Please take a look at our breast tumor decision tree below (one of possible results).

Decision trees are very easy to implement and represent, they do not need a lot of parameters, and they require little data preprocessing.

One of the big advantages is that graphical visualization is possible and allows you to better understand certain errors (see the graph above). However, the problem is that a slight bias in the data can easily corrupt the response. We must also pay attention to the depth of the tree, as for the K of KNN, because the algorithm can quickly become underfitted (too small depth) or overfitted (too huge depth).

Now we can talk about the random forest. It is simply several decision trees then at the end we average the results of all the trees. Yes it is that easy.
